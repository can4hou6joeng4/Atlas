# Security Policy

## Supported Versions

Security fixes target the current `main` branch and the latest published
release, when a release exists.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub private vulnerability reporting if it is available on the repository,
or contact the maintainer privately through the GitHub profile listed on the
repository.

Include:

- Affected version or commit.
- Steps to reproduce.
- Impact and any known prerequisites.
- Whether local files, credentials, macOS permissions, or update delivery are
  involved.

Atlas is a local-first app, so reports involving local session data,
filesystem permissions, Sparkle update delivery, release signing, or bundled
third-party code are especially relevant.
